# Weekly Review

Without a regular cadence, experiments stall, metrics drift, and momentum dies. The weekly review is your self-imposed accountability mechanism — the replacement for the growth partner meeting you'd have in an accelerator.

## The Weekly Review Format

Run this every week, same day, same time. Takes 30-60 minutes.

### 1. Metrics Check (5 min)
- Update your MIK and AARRR metrics
- What moved? What didn't? Any surprises?

### 2. Experiment Review (10 min)
- What experiments ran this week?
- Fill out Learning Cards for completed experiments
- What did you learn? How does it change your assumptions?

### 3. Assumption Update (5 min)
- Did any assumptions get validated or invalidated?
- Has the priority order changed?
- Are there new assumptions to add?

### 4. Next Week Planning (10 min)
- What's the riskiest remaining assumption?
- What's the cheapest experiment to test it?
- Fill out a Test Card for next week's experiment

### 5. Gut Check (5 min)
- Am I still focused on one segment, one product, one value prop?
- Am I building too much before validating?
- What's the one thing that would move the needle most this week?

## What to Track Weekly

Keep it simple. A markdown table is fine:

```

|------|-----------|-----------|---------------|-------------|-----------------|

```

## Red Flags

Watch for these patterns:
- **No experiments run for 2+ weeks** — You're either building too much or avoiding validation
- **MIK hasn't moved in 3+ weeks** — Something fundamental isn't working
- **You keep testing the same assumption** — You might be avoiding the result
- **All experiments "succeed"** — You're probably not testing hard enough
- **You've added features but not customers** — Classic builder trap

## Monthly Retro

Once a month, zoom out:
- Is your goal still the right goal?
- Has your understanding of the customer changed significantly?
- Do your W3 answers still hold up? Should you update them?
- Are you in the same stage or have you progressed?

**Use `/stage` to reassess where you are if things have changed significantly.**
